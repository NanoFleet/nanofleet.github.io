# Installation (Standalone)

Run a single NanoFleet agent with Docker — no dashboard needed. Ideal for simple setups or testing.

## Docker Compose

Create a `docker-compose.yml`:

```yaml
services:
  agent:
    image: ghcr.io/nanofleet/nanofleet-agent:latest
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./workspace:/workspace
    ports:
      - "4111:4111"
```

Copy the example env file and fill in the required values:

```bash
cp .env.example .env
```

Start with:

```bash
docker compose up -d
```

The agent is now running at `http://localhost:4111`.

## With a Telegram channel

Add the Telegram channel service to `docker-compose.yml`:

```yaml
services:
  agent:
    image: ghcr.io/nanofleet/nanofleet-agent:latest
    restart: unless-stopped
    env_file: .env
    volumes:
      - ./workspace:/workspace

  telegram:
    image: ghcr.io/nanofleet/nanofleet-channel-telegram:latest
    restart: unless-stopped
    environment:
      AGENT_URL: http://agent:4111
      TELEGRAM_BOT_TOKEN: ${TELEGRAM_BOT_TOKEN}
      ALLOWED_USERS: ${ALLOWED_USERS}
      NOTIFICATION_USER_ID: ${NOTIFICATION_USER_ID}
    depends_on:
      - agent
```
## Upgrade

```bash
docker compose pull
docker compose up -d
```

Your workspace data is preserved because it lives in the `./workspace` volume mount.
